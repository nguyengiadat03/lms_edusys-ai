## 📝 Documentation Standards

### File Naming

- Use UPPERCASE-WITH-DASHES.md for important docs
- Use lowercase-with-dashes.md for regular docs
- Use README.md for directory documentation

### Content Structure

- Start with title and metadata
- Include table of contents for long docs
- Use clear headings and sections
- Include examples and code snippets
- End with next steps or questions

### Markdown Style

- Use proper markdown formatting
- Include code blocks with language tags
- Use tables for comparisons
- Use checkboxes for checklists
- Use emojis sparingly for visual cues
